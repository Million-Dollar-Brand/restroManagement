# Plan to Fix Missing Staff Import in Orders Endpoint

## Problem Analysis
The error `NameError: name 'Staff' is not defined` occurs in the `create_order` function in `app/api/v1/endpoints/orders.py` because the `Staff` model is not imported.

## Root Cause
The import statement on line 7 only imports `Order, OrderItem, MenuItem, OrderStatus` from `app.models.tenant`, but the `create_order` function uses `Staff` on line 107 to look up the current user by username.

## Solution Plan
Add `Staff` to the existing import statement from `app.models.tenant`.

## Implementation Details
- File: `app/api/v1/endpoints/orders.py`
- Change: Update line 7 to include `Staff` in the import list
- Verification: Test the `/api/v1/orders` POST endpoint to ensure it works

## Risks
- Low risk: This is a simple import addition that doesn't change functionality
